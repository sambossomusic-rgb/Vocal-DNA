# VOCALDNA CONSTITUTION

## Mission

VocalDNA exists to make performing easier.

Every feature must save time, reduce effort, improve decision making, or increase performance quality.

VocalDNA is not a music database.

It is a Performance Intelligence System.

---

## CORE PRINCIPLES

1. Enter information once.
Reuse it forever.

2. Every interaction should reduce future effort while increasing future accuracy.

3. Learn from the performer.

Never force the performer to adapt to the software.

4. The software should become more intelligent after every rating, every gig and every correction.

5. Predictions are suggestions.

The performer is always the final authority.

6. Speed beats beauty.

Fast workflows are more valuable than attractive interfaces.

7. The application should require the minimum number of taps possible.

8. Batch operations should always exist where repetitive work exists.

9. Never ask for information that can be predicted.

10. Every statistic must eventually help answer one practical question.

---

## PRIMARY GOALS

- Find the best key for every song.
- Learn the performer's voice.
- Reduce vocal fatigue.
- Reduce decision fatigue.
- Build better setlists.
- Remember everything.
- Help prepare for gigs.

---

## DESIGN PHILOSOPHY

Simple.

Fast.

Reliable.

Touch friendly.

Offline first.

No unnecessary complexity.

---

## SUCCESS

The software should eventually know the performer well enough that adding a new song requires almost no manual work.
